Outreachy: How a website concept moves to prototype?

Introduction

Outreachy is a non-profit organization that offers a three-month internship for software engineers. Their mission “supports diversity within free and open source software” by working with women, people who are transgender, and people of color.

Design Problem

Outrechy has discovered that many applicants start but do not complete the application. The current website uses a lot of text to define application tasks that are confusing and inefficient to the user. Our goal is:

How can Outreachy’s interface be more intuitive so more applicants complete their internship application?

Ideation

This internship program has a six-week application process where applicants complete these steps:

  • Eligibility form
  • Four essay questions
  • Final application form
  • Choose/contribute to an open source project

To make sure those tasks were included in the prototype, I conducted a brainstorming session to define the tasks needed to complete the application process. The column on the right is the categories that will be included for the sketch project. Input

  • Eligibility screening: The user needs to answer the eligibility questions so the program can serve the population they have identified.
  • General application: The user will provide contact and demographic information so Outreachy can identify and communicate to potential interns.
  • Essay questions: The user needs to answer questions so they can tell their story and how it connects with the program’s mission.
  • Select open-sources project: The user needs to identify which project they can best support based on their experience.
  • Open-source project contribution: The user will complete tasks for an internship project to make sure the user’s skill set is inline with the project’s goals.

Output

  • Due dates: The user needs to plan ahead and identify how this process will fit into their everyday life.
  • Frequently asked questions: The user is working with a new system. A help tool should have commonly known questions so the user can move forward with the application.
  • Essay ideas/themes: The essay questions are personal statements. By providing essay ideas, they will allow the user to be confident when competing their essays.
  • Project descriptions: The user will be able to make a knowledgeable decision about wanting to work on this project for three months.
  • Application checklist/review: The user will be able to confirm that all needed tasks for the application are complete.

Recommendations

  • The first time users will immediately find the “Sign-up” button. It is important to have this button in the top or middle of the page so users can immediately start the application. There could also be “Sign-up” or “Log-in” buttons on this page to only provide the user two options. The “Sign-up” button will take the user to the sign up form while the “Log-in” button will present log in fields when clicked.

  • The “eye” icon, that shows text of the password when written, was used to gain password requirements (i.e. 8 characters, 1 upper case…). To make a distinction between two tasks, the password requirements will be placed to the right of the password fields. When each requirements is made then it changes to green to signify the password requirements are met
    In addition, the “eye” icon was hard to view in the prototype. This may be the reason why the icon was used incorrectly by the user. It is necessary that all icons have the same size so everyone can view and use them.

  • The size of the buttons should be cohesive to size so users can differentiate between different tasks on the screen.

  • The various screens need to have a title at the top of the page. The go back “x” icons are known for phone screens while an “->” icon may be more appropriate for computer screens when navigating from page to page on the wusebsite.

Conclusion

The prototypes tested had an overall success rate of 90%. In addition, we debriefed test participants who felt confident to start and sign-up when apply for the internship. We also observed that the minimal design of the UI can be a plus when making sure applicatants complete the application. It was this type of design that will allow users to be independent to complete their applicattion but will have a help tool so they can ask questions along the way.
