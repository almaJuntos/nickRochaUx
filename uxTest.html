Hipmunk: How can user testing increase the usability of the website?

There are many travel websites that help the user find amazing prices when planning their next vacation or business trip. However, there are different types of travelers based on travel criteria and purpose. Hipmunk is a travel website that allows users to book flights, hotels, and rental cars. It finds travel options that are easy to understand based on user need and what they are purchasing. The Hipmunk mantra is, “Don’t let travel planning drive your nuts!”

Design Problem

Can frequent travelers use Hipmunk to plan their next trip?


Let Hipmunk set you FREE!

Methods

The frequent travelers were users who purchased travel accommodations in the last year and have not used Hipmunk. The recruitment population was created from travel networks by sending out text messages with the following criteria questions:

  • Have you ever used Hipmunk to purchase travel accommodations?
  • How often did you travel in the last year?

The population included two participants who were diverse in travel frequency, complexity of travel criteria, and comfort to making travel plans online.

The test used the following tools:

  • User test script – It maintains a protocol for communication between moderator and users during the test.
  • Pre-test questionnaire - It identifies overall user needs to travel and using online travel services.
  • Task instructions – It gathers data about how users navigate Hipmunk website.
  • Debrief questionnaire – It identifies website improvements while gaining overall feedback about the website after use.

The test used the following data analysis tools:

  • Video recording
  • User test logging sheet
  • System usability scale (SUS)
  • Affinity wall analysis
  • Time mapping of user task

Findings + Recommendations

Summary of Results

The user test observed participants completing six tasks using Hipmunk website. The overall findings are based on function view ability and memory tax overload when using the site. The recommendations for key findings are directed to interface content and function placement.

Key Findings

Finding 1:

In task 1, the users were not able to find the email button on the Hipmunk website to send travel itinerary to other parties. Once the users selected a complete trip, they are directed to another tab away from the Hipmunk website (See figure 1). The users thought they needed to fully purchase the ticket before having an email option based on past travel experiences on the web.

userMunk1:

“I need to book it then send the email.”

userMunk2:

“Where is the send email function?”


Figure 1: The users are directed to a different tab where they purchase the ticket from the airline company website.

Figure 2: The Hipmunk website places the email itinerary function on the top right of the flights results page.

Recommendation:

“Matching between system and real world (Nielson, 1994), “ is the ability to use common logic to complete user tasks within the interface. A development change to the website will allow the user to remain on the Hipmunk site after finding a complete flight. The email itinerary function should change font size and color so the function is more visible to the user.

Finding 2:

During task 4, the user was tasked to find how many trip options are found within a specific time frame. The Hipmunk website provides a time gage function that allows the user to complete the task (See figure 3). Munk1 was able to use the gage function while Munk2 did not understand its overall purpose. However, in both cases, the users were unable to find the selected number of trips content (See figure 4) without scrolling down Hipmunk website and counting the total flight options by hand.

userMunk2:

“I am starting to get confused!”

Figure 3: This is the time gage function on the Hipmunk website.

Figure 4: The Hipmunk website provides a total of flights selected on the top left of the page.

Recommendation:

The number of trips selected content is not viewed by the user in regard to location and font color. By using Gestalt properties to associate like items, we can change the number count to green as seen in Figure 4. The location will also benefit user visibility by placing this content below the “Sort by” drop down menu.

Finding 3:

The Hipmunk website allows the user to compare other travel websites to price. The main screen of the website gives the user different travel sites after selecting your type and time of travel (See figure 5). However, after selecting the search button, the other travel sites open in a completely different window that is minimized and hard to locate (See figure 6).

Figure 5: The Hipmunk website compares flights to other sites.

Figure 6: The Hipmunk website displays the travel site comparisons in a new window at the bottom, right corner of the website.

Recommendation:

Using “aesthetic and minimal design (Nielson, 1994),” it will allow the user to see important content. The travel site comparison function needs to be created within the Hipmunk website. The design will take a tab browser model that is next to the other selected travel fairs in Hipmunk.

Limitations

The limitations of the test are related to population size and possible bias from the participants. These limitations were found based on time and first time learning conditions. Here are some suggestions for future user test for Hipmunk

  • The participant group should be 5 – 10.
  • The participant should adhere to the specific target population.
  • The participants should be independent from the user test moderator.
  • The user test tools (ex. computer type) should be consistent with every participant.

Conclusion

The user test explored how frequent travelers navigate the travel website, Hipmunk. The test used pre-test questionnaire, six user tasks, and a debriefing session with each user. The key findings were based on function view ability and memory tax overload when using the site. The recommendations for key findings were directed to interface content and function placement. The overall usability from the participants shows an average response.

userMunk1

“ This is pretty user friendly.”

userMunk2

“It shows cheaper airfare.”

The people who use travel websites are looking for an experience that is easy to view, have transparent content, and have the ability to shop around with other travel sites. The Hipmunk website is in a good place to amend the minor key findings that will further support ongoing users.
